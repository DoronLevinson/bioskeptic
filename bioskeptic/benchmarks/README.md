# BioSkeptic — evaluation harness

A small, honest evaluation set for **comparing the refuting mechanisms** — which catch the most wrong
claims, and which mis-fire (wrongly refute a working drug) least often. It is a triage tool, not a
gold-standard benchmark; read the caveat at the bottom before quoting any number.

Two files, same schema, regenerated by [`build.py`](./build.py):

- **`drug_disease.jsonl`** — 100 approved drug→disease pairs (TRUE) + 150 easy negatives.
- **`target_disease.jsonl`** — the same 100 collapsed to target→disease (TRUE) + 50 ClinGen refuted
  negatives + 150 easy negatives.

## Where every row comes from

| Rows | Source | Label | Role |
|---|---|---|---|
| Approved drug→disease + target + direction | **Open Targets** `drug()` (approved indications + mechanism-of-action) | TRUE | **mis-fire / precision** — any firing here is wrong |
| Refuted / disputed gene→disease | **ClinGen** Gene–Disease Validity CSV | FALSE (`refuted_curated`) | real, independent negatives |
| Disease-swaps of the TRUE pairs | derived | FALSE (`constructed_disease_swap`) | easy negatives — breaks the target–disease link |
| Direction-flips of the TRUE pairs | derived | FALSE (`constructed_direction_flip`) | easy negatives for direction/dosage mechanisms |

RepoDB was dropped: its host is dead, and its only unique value (failed-trial negatives) is not in this
simplified set. Approved pairs come from Open Targets, which also gives the target and direction natively.

## Row schema

```json
{"id","kind","drug":{"chembl","name"}|null,"target":{"ensembl","symbol"},
 "disease":{"efo","name"},"direction":"inhibit|activate|null","relation",
 "label":true|false,"negative_type","source","source_version","pubmed_bin"}
```

## How to regenerate

```bash
uv run python -m bioskeptic.benchmarks.build   # ~1 min, cached; writes both .jsonl files
```

Deterministic (`SEED = 0`). Sizes are the `N_*` constants at the top of `build.py`. The generated
`.jsonl` files are committed so the eval runs offline and the numbers are stable.

## Sources & versions

- **Open Targets** GraphQL, `api.platform.opentargets.org/api/v4` — queried 2026-07-12.
- **ClinGen Gene–Disease Validity**, `search.clinicalgenome.org/kb/gene-validity/download` — file
  created 2026-07-12, saved in `sources/clingen_validity.csv`.

## How to read the results

Score each mechanism on three columns, kept separate (never merged into one accuracy):

- **mis-fire rate** on the TRUE set — *lower is better* (the safety number).
- **catch rate** on ClinGen negatives — real, subtle.
- **catch rate** on scrambles — easy; a coarse sanity + volume signal.

The leaderboard falls out: sort by catch rate, filter to low mis-fire rate.

## The honest caveat

The scramble negatives are **random and easy**, so catch-rate on them measures *relative* discrimination
between mechanisms, **not** real-world precision on the actual wrong edges a knowledge graph produces.
The ClinGen slice is the one that keeps a foot in "real." `pubmed_bin` (known-vs-obscure stratification)
is stubbed `null` for now — add it when we want the head-to-head-vs-Claude difficulty split.
